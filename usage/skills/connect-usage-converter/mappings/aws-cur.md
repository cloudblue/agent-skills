# AWS CUR → Connect Usage File — Field Mapping

> **Quick reference.** The authoritative version lives on the Connect MCP server — call `get_vendor_cookbook(vendor="aws-cur")` for the up-to-date copy. Use this file for at-a-glance lookups while reasoning about a conversion.

AWS records are treated as **dynamic items** — `item_name` / `item_unit` / `item_mpn` / `item_precision` are populated on every row so Connect can register items it hasn't seen before. The per-row hourly granularity from CUR is folded into `record_note` for traceability; the time fields hold the file-level billing window.

| Connect destination | AWS source (or derivation) |
|---|---|
| **Records sheet** | |
| `record_id` | autogenerated GUID (one UUID4 per record) |
| `record_note` | synthesized: `aws-{line_item_usage_account_id}-{line_item_product_code}-{YYYYMMDDHH}-{row_index}` |
| `item_search_criteria` | literal `"item.mpn"` |
| `item_search_value` | `line_item_product_code` |
| `quantity` | literal `1` |
| `start_time_utc` | first day of the billing month, `00:00:00` UTC (same on every row) |
| `end_time_utc` | last day of the billing month, `23:59:59` UTC (same on every row) |
| `asset_search_criteria` | literal `"parameter.account_id"` |
| `asset_search_value` | `line_item_usage_account_id` |
| `amount` | `line_item_unblended_cost` |
| `category_id` | derived = `AWS.{product_product_name}` |
| `tier` | `0` for Customer, `1` for Reseller |
| `item_name` | `product_product_name` (always — dynamic item) |
| `item_unit` | literal `"Unit"` |
| `item_mpn` | `line_item_product_code` |
| `item_precision` | literal `"Decimal(8)"` |
| **Categories sheet** | |
| `category_id` | `AWS.{product_product_name}` for each distinct `product_product_name` seen in records |
| `category_name` | `product_product_name` |
| `category_description` | e.g. `"AWS {product_product_name} usage"` |
| **Usage File header** (`manage_usage_file` parameters) | |
| `name` | caller's choice, e.g. `"AWS CUR {year}-{month}"` |
| `product_id` | caller-supplied (the Connect product representing AWS) |
| `contract_id` | caller-supplied (partner's distribution contract) |
| `period_from` | first day of billing month, `00:00:00` UTC |
| `period_to` | last day of billing month, `23:59:59` UTC |
| `currency` | inferred from the CUR export (CUR is single-currency per export) |
| `external_id` | optional — `bill_payer_account_id` for traceability |
| `note` / `description` | optional, free text |

## AWS CUR columns that don't write to any Connect field

- `bill_billing_entity` — constant `"AWS"`, redundant with the product selection.
- `bill_payer_account_id` — payer, not consumer; only optionally used as the file's `external_id`.
- `line_item_usage_start_date` — hourly timestamp; preserved in `record_note`, not in the time fields.
- `line_item_line_item_description` — rate description; do **not** copy into `record_note` (would make the invoice too granular).
- `line_item_line_item_type` — used as a *filter* (keep `Usage` / `Fee` / `RIFee`, drop `Tax` / `Credit` / `Refund`), never written to any column.
